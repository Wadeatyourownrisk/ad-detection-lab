# ADR-0001: <Use ADR for Decisions>

**Date:** 2026-04-17 
**Status:** Accepted 
**Phase:** Pre-phase start

## Context

When starting this project I realized the scale of what I was trying to do and the sheer volume of things I was going to have to document and I knew then that I had to standardize some things or I would get lost.


## Options Considered

Creating my own standard was the only real other option. Many of the other standards I saw seemed to center around input collection for decisions and other things that aren't cleanly "why I made this decision"

## Decision

Since it is just me I decided to go with adr because clear documentation is vital to the purpose of this lab. The two goals of this lab are to demonstrate technical capabilities and the ability to properly upkeep documentation.
Creating my own documentation standard would bloat the scope of this project and have minimal value in the long run.

## Consequences

I have to be careful to actually follow adr standards.

## Revisit if

I will revisit this if I run into unforeseen issues or learn something that substantially reframes my approach.
